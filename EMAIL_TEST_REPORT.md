# Отчет о тестировании отправки проверочных кодов на email

## Дата тестирования: 2026-01-13

## Результаты тестирования

### ❌ Отправка проверочных кодов НЕ РАБОТАЕТ

### Проблема
Не удается установить соединение с SMTP сервером `mail.nic.ru` на портах:
- **Порт 465 (SSL/TLS)**: Timeout - сервер не отвечает
- **Порт 587 (STARTTLS)**: Timeout - сервер не отвечает

### Текущие настройки
```
Email: Bot@yourdomain.ru
SMTP сервер: mail.nic.ru
Порт: 587 (последняя попытка)
Use TLS: false
Timeout: 30 секунд
```

### Возможные причины

1. **SMTP доступ не включен**
   - В панели управления nic.ru для почтового ящика `Bot@yourdomain.ru` может быть отключен SMTP доступ
   - Нужно включить SMTP в настройках почтового ящика

2. **Неправильный SMTP сервер**
   - Для домена `yourdomain.ru` может быть нужен другой SMTP сервер
   - Возможные варианты:
     - `smtp.hosting.nic.ru`
     - `mail.hosting.nic.ru`
     - `smtp.yourdomain.ru` (если настроен)
     - Другой сервер, указанный в панели управления

3. **Блокировка файрволом**
   - Провайдер или файрвол может блокировать исходящие подключения на порты 465/587
   - Нужно проверить настройки файрвола

### Что нужно сделать

1. **Проверить панель управления nic.ru:**
   - Войти в панель управления доменом `yourdomain.ru`
   - Найти раздел "Почта" или "Email"
   - Открыть настройки почтового ящика `Bot@yourdomain.ru`
   - Найти раздел "SMTP настройки" или "Настройки отправки"
   - Убедиться, что SMTP доступ **включен**
   - Скопировать правильный SMTP сервер и порт

2. **Проверить документацию nic.ru:**
   - Посмотреть официальную документацию по настройке SMTP для доменов
   - Возможно, для доменов на nic.ru нужны другие настройки

3. **Альтернативные варианты:**
   - Использовать другой почтовый сервис (Yandex, Gmail, Mail.ru)
   - Использовать API для отправки писем вместо SMTP
